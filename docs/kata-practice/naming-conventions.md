# Kata Naming Conventions

Names should help a future reader match the kata statement, solution, and test
without opening every file.

## Review prompts

- Use the kata domain term when it is shorter than a generic helper name.
- Prefer method names that describe the returned value.
- Test names should state the input shape and expected behavior.
- Avoid abbreviations unless the kata statement uses them.
- Keep package names aligned with the source or difficulty group.

## Local check

Rename tests before adding comments that only explain unclear names.
