# Kata Selection Flow

A steady kata journal benefits from mixing difficulty, topic, and maintenance
work instead of only adding new solutions.

## Selection prompts

- Choose one kata that exercises a weak data structure.
- Choose one kata that improves an existing package or test group.
- Prefer a small refactor when the next kata would duplicate helpers.
- Keep source attribution and constraints close to the solution.
- Add follow-up notes when a kata suggests a broader pattern.

## Review prompt

The next kata should improve either coverage, variety, or clarity.
